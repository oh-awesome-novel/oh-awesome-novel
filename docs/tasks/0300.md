Implementation Notes:

- Read tools are exposed through AI SDK `ToolSet` from `packages/tools`.
- Runtime receives and executes `ToolSet` directly; no second `StoryTool` or `RuntimeToolRegistry` abstraction exists.
- `chapter.get` rejects `0001/0000` because `0000.md` is volume metadata, not narrative chapter content.
- Agent default tool assembly includes completed M5 read tools and M6 write-intent tools only; future/proposed tools stay out of the default ToolSet.

## Goal

暴露只读领域工具，让 Runtime 可以通过 Vercel AI SDK `ToolSet` 读取小说工程状态。

这里的 “Tool Registry” 是领域能力集合的概念名，不是第二套自研工具抽象。实现应以 AI SDK `tool()` / `jsonSchema()` 和 `ToolSet` 为准。

## Deliverables

- `createReadTools()`。
- AI SDK `ToolSet`。
- 基于 AI SDK `tool()` / `jsonSchema()` 的 read tools。
- 如 UI 后续需要 `readOnly`、`risk`、`allowedInSkills` 等信息，只允许在 `ToolSet` 外围增加薄 metadata map。
- Read tool result schema。
- Tool call log shape。

## Done Criteria

- read tools 返回结构化数据。
- `packages/tools` 可以创建 AI SDK `ToolSet`。
- Runtime 可以调用 read tools。
- tool call log 可用于 UI 展示。
- 不需要 `defineStoryTool()`、`StoryTool` 或独立 `RuntimeToolRegistry`。

## Constraints

- read tools 不写任何文件。
- tools 命名使用 `domain.action`。
- 不把业务逻辑藏进 Skill prompt。
- 不定义第二套 Tool 抽象来替代 AI SDK `ToolSet`。
