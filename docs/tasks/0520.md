# 0520 HTTP Backend SSE And AI SDK Vue Interface

> Status: Planned
> Milestone: M8 Minimal Copilot Interface

## Goal

实现一个本地 HTTP backend，让前端通过 HTTP SSE 获取 agent 流式消息。

在 `packages/agent` 层增加 Vercel AI frontend compatibility，输出可被 `@ai-sdk/vue` 直接消费的 UI message stream / SSE 协议形态。

## Preferred Shape

以 HTTP backend 作为 Electron renderer、Web panel、后续调试页面的统一接入层。

```text
Vue frontend
  -> @ai-sdk/vue
  -> HTTP POST /api/agent/chat
  -> SSE stream
  -> backend
  -> packages/agent streamNovelAgentTurn()
  -> packages/runtime RuntimeEvent stream
  -> packages/tools AI SDK ToolSet
  -> filesystem
```

Electron 版不优先实现一套独立 UI stream。Electron main process 负责启动本地 backend，再把 backend base URL 交给 Vue renderer。

## Subtasks

### 0520A HTTP Backend And SSE Transport

Deliverables:

- 实现本地 HTTP backend。
- 提供 agent chat endpoint。
- endpoint 使用 SSE 返回流式消息。
- SSE 必须能持续输出 assistant text delta，避免 UI 看起来卡死。
- SSE 必须携带 tool call、tool result、pending action 等可展示事件。
- backend 绑定 `127.0.0.1`，默认使用随机可用端口。

Done Criteria:

- 前端可以通过 HTTP 请求启动一次 agent turn。
- 前端可以边生成边接收消息。
- tool log 和 pending action 能从流中重建。
- backend 不直接绕过 Human Approval 写入文件。

### 0520B Vercel AI Frontend Compatibility Layer

Deliverables:

- 在 `packages/agent` 暴露 RuntimeEvent 到 Vercel AI UI stream 的适配器。
- 兼容层面向 `@ai-sdk/vue`，但不要在 runtime 中引入 Vue 或 frontend 依赖。
- 保留项目自己的 RuntimeEvent 作为内部边界。
- backend 只负责 HTTP/SSE 包装，协议转换逻辑优先复用 agent 层适配器。

Done Criteria:

- Vue frontend 可以直接用 `@ai-sdk/vue` 接入 agent chat endpoint。
- `packages/runtime` 仍只输出 RuntimeEvent stream。
- `packages/agent` 不重写 tool loop。
- `packages/tools` 仍只暴露 AI SDK `ToolSet`。

### 0520C Vue Agent Conversation Checkpoint

Deliverables:

- 使用 Vue 和 `@ai-sdk/vue` 实现一个最小 agent 对话闯卡页面。
- 闯卡用于验证 agent 基础能力，而不是完整 IDE。
- 页面至少展示 streaming assistant message、tool activity、pending action summary。
- 关卡建议：
  - Level 1：读取 workflow / constitution。
  - Level 2：读取 character / state。
  - Level 3：触发一个 write intent，展示 pending action，不自动写入。

Done Criteria:

- 开发者能用该页面快速验证 agent 是否能流式输出。
- 开发者能看到 tool call 发生顺序。
- write intent 只停在 pending action，不绕过审批。

### 0520D Electron Composition

Deliverables:

- Electron main process 启动和关闭 HTTP backend。
- Electron renderer 使用 Vue frontend。
- renderer 通过 main process 提供的 backend base URL 访问本地 backend。
- 生产形态下不要求 renderer 直接访问文件系统。

Done Criteria:

- 启动 Electron 后，Vue frontend 可以连接本地 backend。
- agent 对话闯卡可以在 Electron 中运行。
- backend 生命周期跟随 Electron main process。
- 没有新增第二套 Electron-only agent streaming 协议。

## Constraints

- 只能把 HTTP backend 作为 transport / composition layer，不把它做成新的 agent runtime。
- 不引入 LangChain、AutoGen、CrewAI、Semantic Kernel 或重型 agent framework。
- 不用 AI SDK `ToolLoopAgent` 替代项目自己的 Aider-style runtime。
- 不在 `packages/runtime` 中注册领域工具。
- 不在 `packages/runtime` 中引入 `@ai-sdk/vue`、HTTP server、Electron 或具体 LLM provider。
- 前端不得直接调用 filesystem、tools execute 或 Apply Engine。
- 所有生产写入仍必须经过 PendingAction、diff preview 和 Human Approval。
