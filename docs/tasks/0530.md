# 0530 Global Workspace Launcher

> Status: Completed
> Milestone: M8 Minimal Copilot Interface

## Goal

软件启动后不直接进入某个小说工程，而是先展示全局工作空间列表，体验类似 WebStorm / JetBrains 启动页。

用户应能从启动页选择一个已有小说 workspace，或通过显式操作导入 / 打开一个 workspace。选择前不启动 workspace 级 agent，不读取小说正文，不展示工作台三栏界面。

导入 workspace 时，必须判断用户选择的目录是否是一个 OAN workspace。当前判定以现有 filesystem-first 设计为准：目录下应存在 `.oan/config.yaml`、`.oan/workflow.yaml`，并符合基础小说工程目录结构。无效目录不能被加入 workspace list。

## Reference Notes

`reference-only/note-gen` 使用本地 store 记住 `workspacePath`、`workspaceHistory` 和 `currentPage`，并在入口页恢复上次页面。这个项目只借鉴“启动入口 + 最近工作区 + 进入工作区”的体验，不照搬 note-gen 的 Tauri store、同步体系或默认 `article/` 工作区。

## Deliverables

- 全局启动页 / launcher。
- 最近工作空间列表。
- 导入已有 workspace 的入口。
- OAN workspace 目录校验。
- 导入失败原因提示。
- 从列表移除 workspace 的入口。
- 当前 workspace 基础信息展示：名称、小说名称、路径、最近打开时间。
- workspace 显示名称编辑入口。
- workspace 列表搜索。
- 空列表状态。
- 无效路径 / 已移动路径提示。
- 进入 workspace 后记录最近使用信息。

## Workspace Import

导入流程：

1. 用户点击导入 workspace。
2. 应用让用户选择一个目录。
3. 应用校验该目录是否是 OAN workspace。
4. 校验通过后加入全局 workspace list。
5. workspace 显示名称默认使用小说名称。
6. 用户可以单独修改 workspace 在 launcher 中显示的名称。

OAN workspace 第一版校验规则：

- 目录存在且可读。
- 目录下存在 `.oan/`。
- 目录下存在 `.oan/config.yaml`。
- 目录下存在 `.oan/workflow.yaml`。
- 至少存在一组小说内容目录，例如 `chapters/`、`characters/`、`world/`、`state/`、`timeline/`、`foreshadow/` 或 `summaries/`。
- 目录不是 `.git/`、`.workspace/`、`.oan/sessions/` 或这些内部目录的子目录。

如果目录不满足校验规则，UI 应说明原因，例如：

- 不是 OAN workspace。
- 缺少 `.oan/config.yaml`。
- 缺少 `.oan/workflow.yaml`。
- 目录不可读。
- 不能导入 workspace 内部隐藏运行时目录。

## Workspace Name

workspace list 中的显示名称默认使用小说名称。

小说名称来源优先级：

1. `.oan/config.yaml` 中的小说名称字段。
2. `.oan/workflow.yaml` 或 constitution 中后续稳定定义的项目名称字段。
3. workspace 目录名。

显示名称是 launcher 层元数据，可以单独修改。修改显示名称不应重命名 workspace 目录，不应修改小说正文文件，也不应改变小说事实源。

## Workspace List Search

workspace 列表应支持搜索，使用 [`lucaong/minisearch`](https://github.com/lucaong/minisearch) 作为本地搜索引擎。

搜索索引只覆盖 launcher 的 workspace list 元数据，不扫描 workspace 小说正文。

建议索引字段：

- `name`: launcher 显示名称。
- `novelName`: 小说名称。
- `path`: workspace 绝对路径。
- `lastOpenedAt`: 最近打开时间，可用于排序或展示，不作为全文字段。

搜索能力第一版应支持：

- 按显示名称搜索。
- 按小说名称搜索。
- 按路径片段搜索。
- 空搜索时按最近打开时间展示。

MiniSearch index 是派生缓存，可以在 workspace list 变更时重建。

## Done Criteria

- 应用冷启动后先展示 workspace 列表。
- 用户选择 workspace 后才进入 workspace 工作台。
- 没有 workspace 时展示清晰的空状态和打开入口。
- 最近 workspace 列表持久化，但不成为小说事实源。
- 用户可以导入已有 workspace。
- 导入时会拒绝非 OAN workspace 目录。
- 导入成功后 workspace 出现在列表中。
- workspace 显示名称默认等于小说名称。
- 用户可以单独修改 workspace 显示名称。
- 修改显示名称不会修改小说目录名或小说内容。
- workspace 列表可以通过 MiniSearch 搜索。
- 搜索 workspace 列表不会读取小说正文。
- 移除最近项不删除真实 workspace 文件。
- 打开不存在或不可读的 workspace 时不崩溃，并允许用户从列表移除。

## Constraints

- 不在 launcher 中运行 agent。
- 不在 launcher 中读取整本小说内容。
- workspace list 搜索只索引 workspace 元数据，不索引小说文件内容。
- 不把 workspace 列表存进小说 workspace 内部。
- 不引入账号、云同步或多设备同步。
- 不允许把非 OAN workspace 加入 workspace list。
- 不允许导入 `.git/`、`.workspace/`、`.oan/sessions/` 等内部目录。
- 修改 launcher 显示名称不得修改小说事实源。
- 不改变 filesystem-first：workspace 本身仍由文件树、Markdown、YAML 和 Git 表达。

## Implementation Notes

- Implemented in `apps/desktop-ui/src/App.vue`, `apps/desktop-ui/src/components/workspace/WorkspaceLauncher.vue`, `apps/desktop-ui/src/composables/useWorkspaceApi.ts`, `apps/desktop-ui/src/composables/useWorkspaceSearch.ts`, and `packages/backend/src/index.ts`.
- Workspace list metadata is persisted in the global `.oan` config directory through the existing core workspace list helpers, not inside any novel workspace.
- Launcher search uses MiniSearch over workspace metadata only: display name, novel name, and path.
- Import validation rejects non-OAN directories and internal runtime paths before adding an entry.
- Verified with `npm run build --workspace @oh-awesome-novel/desktop-ui`, `npm run test:run --workspace @oh-awesome-novel/test-backend`, and browser smoke coverage for empty launcher, invalid import, valid import, and opening a workspace.
