# 0540 Workspace Entry LLM Provider Gate

> Status: Completed
> Milestone: M8 Minimal Copilot Interface

## Goal

用户从全局 workspace 列表选择某个 workspace 后，如果当前应用没有可用的 LLM provider 配置，则先弹出配置窗口。

Provider 配置完成或用户明确跳过后，才进入 workspace 工作台。进入工作台后，未配置 provider 时仍可浏览文件和查看纯文本内容，但 Copilot 入口应显示不可用状态，并提供配置入口。

## Deliverables

- Workspace entry provider 检查。
- LLM provider 配置弹窗。
- provider 类型 / base URL / model / API key 等基础字段。
- 保存、取消、稍后配置流程。
- 配置验证结果展示。
- 配置完成后继续进入所选 workspace。
- 未配置 provider 时的 Copilot 禁用态。
- 从 workspace 工作台重新打开 provider 配置的入口。

## Done Criteria

- 选择 workspace 后会检查全局 provider 配置。
- 没有 provider 配置时弹出配置窗口。
- 保存有效配置后进入 workspace。
- 用户取消或稍后配置时仍可进入只读工作台。
- 未配置 provider 时不会发起 agent 请求。
- API key 等敏感配置不写入小说 workspace。
- 配置失败或验证失败时提示原因，不静默吞错。

## Constraints

- Provider 配置属于应用级设置，不属于小说事实源。
- 不把 API key 写入 Markdown / YAML / Git 管理的小说文件。
- 不要求首次进入必须联网验证。
- 不在 provider 配置流程中自动生成或修改小说内容。
- 不绕过现有 core provider config 设计。

## Implementation Notes

- Implemented in `apps/desktop-ui/src/App.vue`, `apps/desktop-ui/src/components/workspace/ProviderGateModal.vue`, `apps/desktop-ui/src/components/workspace/CopilotPanel.vue`, and `packages/backend/src/index.ts`.
- Provider config is checked when a workspace opens; missing config shows the provider gate while still allowing explicit read-only entry.
- Saved provider config is persisted as application-level global config in `llm-providers.json` under the resolved global `.oan` config directory.
- The UI stores API key environment variable names only; no API key is written into Markdown, YAML, Git-managed novel files, or the novel workspace.
- Verified with `npm run build --workspace @oh-awesome-novel/desktop-ui`, `npm run test:run --workspace @oh-awesome-novel/test-backend`, and browser smoke coverage for provider gate and read-only entry.
