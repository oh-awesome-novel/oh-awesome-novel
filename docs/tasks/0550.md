# 0550 NoteGen Inspired Workspace Shell

> Status: Completed
> Milestone: M8 Minimal Copilot Interface

## Goal

进入某个 workspace 后，展示类似 `reference-only/note-gen` 的工作界面：

```text
顶部：workspace toolbar
左侧：workspace 文件列表
中间：文件内容 / workspace home
右侧：Agent Copilot
```

第一版只做小说 workspace 浏览与 Copilot 承载，不做 Markdown / YAML 富渲染，不做手动编辑器，不照搬 note-gen 的数据库、同步、标签、富文本编辑器或移动端复杂功能。

## Reference Notes

note-gen 的主界面使用可调整宽度的三栏布局：左侧 `LeftSidebar` / `FileSidebar`，中间 `EditorLayout`，右侧 `Chat`。它还维护 `activeFilePath`、`openTabs`、文件树、workspace history 等状态。

本项目借鉴：

- 三栏工作台信息架构。
- 顶部工具栏承载全局操作。
- 左侧文件树作为 workspace 导航。
- 单独的章节导航视图作为小说正文导航，避免只依赖 `0001.md` 这类稳定编号文件名。
- 中间区域随选中文件变化。
- 右侧 Copilot 作为辅助面板。
- 面板可折叠 / 可调整宽度的体验。

本项目不借鉴：

- Tiptap Markdown 编辑器。
- 图片、音频、截图、OCR、同步、向量数据库、标签笔记等 note-gen 特有功能。
- 直接编辑文件内容。

## Deliverables

- Workspace 顶部 / 底部基础状态信息：workspace 名称、路径、provider 状态。

## Top Toolbar

Workspace shell 顶部需要有一个常驻 toolbar，承载当前 workspace 的全局操作。

第一版 toolbar 至少包含：

- 全局搜索入口，连接 `0570 Workspace Global Search`。
- 展开 / 收起左侧文件列表。
- 展开 / 收起右侧 Copilot。
- 设置入口。
- 当前 workspace 名称或路径的简短显示。
- 章节导航入口，连接 `0555 Chapter Navigation View`。

可选快捷按钮：

- 回到 workspace home。
- 查看 PendingAction。
- 查看 Git dirty 状态。
- 退出 workspace 回到 launcher。

Toolbar 中的按钮应只触发显式 UI 状态变化或打开对应面板 / 页面，不应静默写入小说 workspace。

## Done Criteria

- 进入 workspace 后默认展示三栏 shell 的基本结构。
- 顶部 toolbar 始终可见。
- toolbar 可以打开全局搜索。
- toolbar 可以展开 / 收起左侧文件栏。
- toolbar 可以展开 / 收起右侧 Copilot。
- toolbar 可以打开设置入口。
- 左侧文件树来自当前 workspace 文件系统。
- 章节导航入口可以打开按卷 / 章节标题组织的正文列表。
- 选择 Markdown / YAML / 普通文本文件后，中间区域以纯文本展示原始内容。
- Markdown 不做渲染，YAML 不做结构化表单展示。
- 中间区域第一版不允许手动编辑。
- 右侧 Copilot 可以承载已有 agent chat / tool log / pending action UI。
- 文件树和内容查看不会读取 workspace 外部路径。

## Constraints

- 不做通用 IDE。
- 不做富文本编辑器。
- 不做多标签编辑器，除非后续任务明确要求。
- 不自动打开上次文件；默认进入 workspace 时不选中文件。
- toolbar 快捷按钮不得静默修改 workspace。
- 全局搜索入口不得默认把搜索结果塞进 agent context。
- 文件树不需要承担章节标题列表职责，章节正文导航由独立章节导航视图承载。
- 不让 frontend 直接执行写工具或绕过 backend / agent。
- 不在 UI 状态中隐藏小说事实源。

## Implementation Notes

- Implemented in `apps/desktop-ui/src/components/workspace/WorkspaceShell.vue`, `FileTreePanel.vue`, `FileTreeNodeItem.vue`, `FileViewer.vue`, `CopilotPanel.vue`, `apps/desktop-ui/src/style.css`, and `packages/backend/src/index.ts`.
- The visual layout follows the `reference-only/note-gen` three-pane desktop pattern: top toolbar, left navigation, center viewer/home, and right Copilot panel.
- File tree and file content are read through backend endpoints only; the frontend never receives arbitrary filesystem access.
- The toolbar Search entry opens a lightweight file-path search over the already loaded tree. Full workspace content search remains scoped to task `0570`.
- Verified with `npm run build --workspace @oh-awesome-novel/desktop-ui` and browser smoke coverage for shell render, panel navigation, Search open, and opening a raw file from Search.
