# 0555 Chapter Navigation View

> Status: Planned
> Milestone: M8 Minimal Copilot Interface

## Goal

因为正文路径采用稳定编号规则，文件树中会反复出现 `0001.md`、`0002.md` 这类文件名。单纯依赖文件视图时，作者很难直接看到“卷 / 章节列表 / 章节标题”的创作结构。

Workspace UI 需要增加一个单独的章节导航视图，用小说语义展示章节，而不是用文件名本身承担阅读导航。

## Data Source

章节导航视图必须从 filesystem-first 的小说工程派生，不引入隐藏数据库：

- 扫描 `chapters/<volume-number>/`。
- 卷目录使用 4 位稳定编号，例如 `0001/`。
- 每卷 `0000.md` 是卷信息 / 卷元数据，不是小说正文章节内容。
- 正文章节从 `0001.md` 开始。
- 章节 id 使用 `<volume-number>/<chapter-number>`，例如 `0001/0003`。
- 卷名优先从 `0000.md` 的 frontmatter / 标题读取。
- 章节标题优先从章节文件 frontmatter / 一级标题读取。
- 标题缺失时使用稳定编号 fallback，例如 `0001/0003 未命名章节`。

修改卷名或章节标题不得导致路径变化。

## Generated Index File

扫描章节后，系统可以生成章节目录索引文件：

```text
.oan/indexes/chapters.yaml
```

索引文件是派生缓存，不是小说事实源。它用于让 UI 快速展示章节目录，并保留上一次扫描结果。

索引文件至少包含：

- 生成时的 Git HEAD commit hash。
- 生成时间。
- 扫描范围。
- 卷 id、卷标题、卷信息文件路径。
- 章节 id、章节标题、章节文件路径。

打开章节导航时必须比较当前 Git HEAD 和索引文件中记录的 hash：

- hash 一致：可以展示索引为当前扫描结果。
- hash 不一致：提示需要重新扫描，不应静默把旧索引当作最新章节目录。
- 无法读取 Git HEAD：提示索引状态未知，并提供重新扫描入口。
- workspace dirty：提示索引可能无法代表未提交修改。

重新扫描是用户显式触发的只读扫描 + 派生索引写入，不是 AI 静默写小说内容。

## User Experience

章节导航视图第一版以只读导航为主：

- 按卷分组展示章节。
- 卷按数字升序排序。
- 章节按数字升序排序。
- 卷标题旁显示卷编号。
- 章节行展示章节标题、章节 id、文件路径。
- `0000.md` 只作为卷信息来源，不作为章节行展示。
- 点击章节后，中间区域打开对应章节文件的纯文本查看器。
- 左侧文件树应同步高亮或可快速定位到对应文件。
- 当前打开章节在章节导航视图中高亮。
- 如果章节索引 hash 过期，展示“需要重新扫描”的状态和重新扫描按钮。

章节导航可以作为左侧文件栏的一个 tab、顶部 toolbar 的一个入口、或 workspace home 的快捷入口。第一版不要求替代完整文件树；文件树仍用于浏览角色、世界观、状态、摘要等其他对象文件。

## Deliverables

- 章节索引读取 API 或 backend endpoint。
- 章节扫描后生成 `.oan/indexes/chapters.yaml`。
- 章节索引 Git HEAD hash 校验。
- 从 `chapters/` 派生的卷 / 章节列表数据结构。
- 卷信息文件 `0000.md` 的只读识别。
- 章节标题读取逻辑。
- 章节导航 UI。
- 章节点击后打开纯文本文件查看器。
- 当前章节高亮。
- 与左侧文件树选中状态的基础联动。
- Workspace toolbar 或 home 中的章节导航入口。
- 空章节状态。
- 章节索引错误状态。
- 章节索引过期状态和重新扫描入口。

## Done Criteria

- 作者可以在 workspace 内打开章节导航视图。
- 章节导航视图不把 `0000.md` 展示成正文章节。
- 相同文件名的不同章节可以通过卷名、章节标题和章节 id 区分。
- 点击章节会打开 `chapters/<volume>/<chapter>.md` 的纯文本内容。
- 章节标题变化不会导致文件重命名。
- 章节列表来自当前 workspace 文件系统。
- 扫描后可以生成 `.oan/indexes/chapters.yaml`。
- 章节导航能识别索引中的 Git hash 与当前 Git HEAD 是否一致。
- hash 不一致时 UI 提示需要重新扫描。
- workspace dirty 或无法读取 Git HEAD 时 UI 不把索引标记为已验证最新。
- 章节导航不会读取 workspace 外部路径。
- 章节导航不会自动触发 Copilot 或写入工具。
- frontend 不直接扫描 filesystem，必须通过 backend / core / tools 的只读能力获取数据。

## Constraints

- 不做手动章节编辑器。
- 不做拖拽重排章节。
- 不做章节重命名写入。
- 不把章节索引保存成事实源数据库；索引只是派生缓存。
- 不让 `0000.md` 进入 chapter.get / chapter summary 等正文章节流程。
- 不用章节标题替代稳定编号路径。
