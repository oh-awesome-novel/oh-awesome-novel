# 0560 Workspace Home Quick Actions And Copilot Visibility

> Status: Planned
> Milestone: M8 Minimal Copilot Interface

## Goal

默认进入 workspace 时，不选中任何文件，右侧 Copilot 隐藏，中间区域展示 workspace home 和快捷操作。

快捷操作面向小说创作工作流，不是营销页。它应该帮助作者快速开始常见动作，例如生成下一章、查看待审批修改、打开 provider 配置、退出当前 workspace 等。

## Initial Home Layout

```text
左侧：文件列表可见
中间：workspace home
右侧：Copilot 隐藏
```

中间 home 第一版包含：

- 当前 workspace 名称和路径。
- Provider 状态。
- 最近 PendingAction / Git dirty 状态摘要。
- 快捷操作：
  - 生成下一章。
  - 打开 Copilot。
  - 查看待审批修改。
  - 打开设置 / 配置 LLM provider。
  - 退出 workspace 回到全局 workspace 列表。

## Deliverables

- Workspace home 空选中状态。
- 不选中文件时隐藏右侧 Copilot。
- 打开 / 隐藏 Copilot 的显式控制。
- 生成下一章快捷操作入口。
- 退出 workspace 回到 launcher 的入口。
- 查看 PendingAction 的入口。
- Provider 未配置时的配置入口。
- 文件选中后中间区域切换为纯文本文件查看。

## Done Criteria

- 默认进入 workspace 时 `activeFilePath` 为空。
- 默认不自动选中第一个文件。
- 默认右侧 Copilot 不占用主要阅读空间。
- 点击“打开 Copilot”后右侧面板出现。
- 点击文件后中间区域展示该文件纯文本内容。
- 点击“退出 workspace”后回到全局 workspace 列表。
- “生成下一章”只发起明确的 Copilot / write-intent 流程，不直接写小说文件。

## Constraints

- Home 不是 landing page，不写营销文案。
- 不在快捷操作中静默修改 workspace。
- 不在未配置 provider 时启动 agent 请求。
- 不把快捷操作实现成隐藏 planner。
- 不做 Markdown / YAML 手动编辑。
