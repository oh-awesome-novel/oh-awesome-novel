# 1010 Context Package And Source Discipline

> Status: Planned
> Milestone: M10 Workflow And Skills

## Related Plans

- [Context Package And Source Discipline](../superpowers/plans/2026-06-19-context-package-source-discipline.md)

## Goal

实现轻量 context source discipline：让 agent 在写作、规划、审稿和整理前能说明读取了哪些 source、为什么读取、哪些省略，以及最小记忆包包含什么。

## Deliverables

- core 类型：
  - `ContextPackage`
  - `ContextSourceRef`
  - `ContextBudgetLayer`
  - `SemanticBoundary`
  - `MinimalMemory`
  - `RuleStackEntry`
- source id registry 的轻量常量，不实现重型 runtime registry
- context package builder helper
- session artifact 写入策略：
  - 普通任务可只保留 assistant visible summary
  - 长任务或需要续跑的任务写 `.workspace/sessions/<id>/context-package.yaml`
- tests 覆盖 selected / omitted / protected / compressible / excluded / L0-L3

## Done Criteria

- 写作相关 command 可以构建 context package。
- context package 明确 selected 和 omitted sources。
- minimal-memory 是 context package 子字段，不是新数据库。
- source id 只服务解释、测试和 session artifact，不替代 tool API。

## Constraints

- 不新增完整 Context Source Registry runtime。
- 不默认全量读取小说 workspace。
- 不把 context package 写入事实域。
