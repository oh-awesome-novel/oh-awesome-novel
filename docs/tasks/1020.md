# 1020 Planning Commands And Prewrite Calibration

> Status: Planned
> Milestone: M10 Workflow And Skills

## Related Plans

- [Planning Commands And Prewrite Calibration](../superpowers/plans/2026-06-19-planning-and-prewrite-workflow.md)

## Goal

实现规划粒度分层：普通单章保持轻量本章契约，卷级 / 大纲级规划承载复杂 gate 和结构化节点。

## Deliverables

- `ChapterContract` 类型与 formatter
- `PlanningPacket` 类型：
  - outline-level
  - volume-level
  - chapter-level
- `/规划大纲` quick command contract
- `/规划下一卷` quick command contract
- `/规划下一章` 输出稳定轻量本章契约
- `/写下一章` 前输出短 `PRE_WRITE_CHECK`
- tests 覆盖普通单章不强制卷级 gate

## Done Criteria

- `/规划下一章` 只默认要求轻量字段：chapter id / title candidate、当前任务、POV、核心冲突、出场角色、hook、章尾变化、禁止事项。
- `/规划大纲` 和 `/规划下一卷` 可以启用读者承诺、伏笔债、关键章节点、CBN / CPNs / CEN 风格拆章。
- `/写下一章` 先给短 `PRE_WRITE_CHECK`，再使用 `chapter.createDraft`。
- 规划产物默认是 session artifact 或 assistant 输出，不自动成为长期事实源。

## Constraints

- 不把完整 `prewrite / precommit / postcommit` 作为每章默认流程。
- 不把本章契约写入 `chapters/` 正文。
- 不跳过 PendingAction。
