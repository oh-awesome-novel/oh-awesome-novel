# 1030 Review And Settlement Workflow

> Status: Completed
> Milestone: M10 Workflow And Skills

## Related Plans

- [Review And Settlement Workflow](../superpowers/plans/2026-06-19-review-and-settlement-workflow.md)

## Goal

把 `/审稿`、`/去AI味`、`/整理本章` 从宽泛 prompt 升级为稳定 schema 驱动 workflow。

## Deliverables

- `ReviewFinding` schema
- review dimension checklist
- `ObservationLog` schema
- `SettlementBundle` schema
- evidence-only settlement prompt section
- hook operation taxonomy：mention / advance / resolve / defer / create
- `/去AI味` 保护规则
- tests 覆盖：
  - review 不隐式 settle
  - settlement 先 observation 后 patch proposal
  - unresolved ambiguity 不直接写 truth files

## Done Criteria

- `/审稿` 默认只输出 findings 和 dimension pass。
- 用户要求改写时，review finding 才能进入 `chapter.createDraft` 或 SemanticPatch 候选。
- `/整理本章` 先输出 observation log，再提出 summary / state / timeline / foreshadow / character PendingActions。
- settlement 只记录正文证据支持的事实。

## Constraints

- 不从大纲或计划中补正文没发生的事实。
- 不自动修改旧 hook，除非正文有证据。
- 不让去 AI 味删除剧情事实、伏笔、钩子、角色特征或必要转折。
