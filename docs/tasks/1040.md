# 1040 Session Artifacts And Author Reports

> Status: Completed

## Related Plans

## Goal

为长任务和需要续跑的 agent run 建立轻量 session artifact，而不是引入复杂 run ledger 数据库。

## Deliverables

- `.workspace/sessions/<session-id>/` 目录约定
- session artifact schema：
  - `run.yaml`
  - `context-package.yaml`
  - `outputs.yaml`
  - `proposed-patches.yaml`
  - `unresolved.md`
- session resume boundary helper
- author report formatter
- tests 覆盖 session artifact 写入、读取、手改检测和续跑提示

## Done Criteria

- 长章节生成、参考拆解、导入、Play adoption 等长任务可记录关键输入 source、输出 artifact、proposed patch list、时间和未决问题。
- 如果用户手改了章节或对象文件，下一轮能提示沿用手改、基于手改继续或放弃旧草稿。
- 最终报告面向作者，不输出原始 JSON、traceback 或长命令日志。

## Constraints

- 第一版不要求每个普通 chat turn 都写完整 run log。
- session artifact 不替代 Git 历史。
- 不把 session artifact 当小说事实源。
