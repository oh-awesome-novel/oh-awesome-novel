## Goal

把已经存在的 `ContextPackage`、runtime tool log、session artifact helper 和 author report 串成端到端 agent run workflow。

当前 core 类型和 helper 已完成，但 model mode 仍主要依赖 workspace snapshot + skill prompt。后续需要让写作相关 run 自动产出 context trace、context package artifact、session artifact 和作者报告。

## Current Code State

- `packages/core/src/agent-context-package.ts` 已扩展 `trace` 字段，并在 model-visible summary 中展示 selected / omitted / trace。
- `packages/agent/src/index.ts` 已提供 `inferNovelAgentCapability()` 和 `createBaselineNovelAgentContextPackage()`，model mode 可基于 quick command / request 分类自动构建 context package。
- `packages/agent/src/index.ts` 已在 long / writing-related / PendingAction run 结束后写 `.workspace/sessions/<session-id>/run.yaml`、`outputs.yaml`、`proposed-patches.yaml`、`unresolved.md` 和 `context-package.yaml`。
- `packages/agent/src/index.ts` 会把 runtime tool log 转成 trace，并把 touched files 写入 resume boundary。
- `packages/backend/src/index.ts` 的 model mode 会把 reference selection 和 project health guardrails 传入 agent intake。

## Implementation Notes

- `ContextPackage.trace` 只记录本轮读取、遗漏、压缩、tool call 和 PendingAction 结果，不保存隐藏推理。
- session store 仍保留 `.oan/sessions` runtime transcript / tool log；规范 author-facing artifact 额外写入 `.workspace/sessions`。
- author report 作为 `outputs.yaml` 中的 assistantText artifact 输出，避免暴露原始 JSON log。
- 覆盖测试：
  - `__test__/core/src/agent-context-package.test.ts`
  - `__test__/agent/src/message-assembly.test.ts`
  - `__test__/agent/src/session-store.test.ts`

## Deliverables

- Extend `ContextPackage` with a lightweight `trace` field.
- Add deterministic context package builder for backend / agent model mode.
- Infer capability from quick command metadata or request classification.
- Map workspace snapshot and tool activity to selected / omitted / trace entries.
- Write `.workspace/sessions/<session-id>/context-package.yaml` for long or write-producing runs.
- Convert runtime result and PendingActions into session artifact files.
- Generate author report after long or write-producing runs.
- Add resume boundary wiring for touched files when a run produces artifacts.
- Tests covering trace, autowiring, artifact writes, and resume prompt behavior.

## Done Criteria

- A `/写下一章` or `/整理本章` run can produce a context package without callers manually passing one.
- The context package records selected sources, omitted sources, and trace entries with reasons.
- Runtime tool calls that read or write relevant files can be reflected in trace or session artifact metadata.
- PendingAction-producing runs write session artifacts under `.workspace/sessions/<session-id>/`.
- Final author report summarizes candidate outputs, pending actions, unresolved decisions, and next step without raw JSON logs.
- Resume check warns when touched files changed after the recorded boundary.

## Constraints

- Do not create a heavy Context Source Registry runtime.
- Do not require every casual chat turn to write full artifacts.
- Do not treat context packages or session artifacts as truth files.
- Do not expose hidden chain-of-thought in trace.
- Keep `packages/runtime` generic and provider-agnostic.
