## Goal

把 `0900` 的 reference bundle 规划收敛出一个可执行的 reference context selector。

Selector 负责在写作、规划、审稿或风格调整时选择少量 distilled reference entries，并明确 included / omitted / reason / budget。它不负责把参考原文直接塞进 prompt，也不负责把参考作品事实写入当前小说 truth files。

## Current Code State

- `ContextSourceId` 已包含 `referenceDistilled`。
- `packages/core/src/reference-work.ts` 已实现 reference import / list / enable / context selector。
- selector 输出 included / omitted / reason / budgetLayer / semanticBoundary / noCopyWarnings / originalSourceRead。
- `formatReferenceContextSelectionMarkdown()` 提供 model-visible distilled-only summary。
- `referenceSelectionToContextSources()` 可把 selector 输出转换为 `referenceDistilled` selected / omitted sources。
- `packages/backend/src/index.ts` 已提供 reference context route，并在 model mode intake 中选择 enabled distilled references。
- `apps/desktop-ui/src/components/workspace/ReferenceImportTab.vue` 已显示 no-copy guardrail、distilled-only 状态、included / omitted reference context。

## Implementation Notes

- selector 默认只读 `examples/references/<id>/context/reference-summary.md`，不会把 `sources/original.*` 放进写作 prompt。
- disabled / explicit mismatch / missing summary / token budget / max count 都会进入 omitted reasons。
- no-copy / differentiation warnings 进入 selector 输出和 model-visible formatter。

## Deliverables

- Reference context selector input / output schema。
- Reader for enabled reference bundle indexes from `examples/references.yaml` or the final 0900 storage layout。
- Selector that returns included distilled entries, omitted references, reasons, budget layer, and no-copy warnings。
- Context package integration using `referenceDistilled` source id。
- Guardrail that original source content is not read by default.
- Formatter for model-visible reference summary that excludes copyable prose.
- Tests for included / omitted / disabled reference behavior.

## Done Criteria

- Agent can explain exactly which reference distilled entries were used and why.
- Disabled references are omitted with reasons.
- Original source files are not loaded unless an explicit user action requests verification or re-deconstruction.
- Selector output includes no-copy / differentiation warnings.
- Reference-derived adoption still requires PendingAction / diff / Human Approval.

## Constraints

- Do not implement full deconstruction pipeline here if `0900` storage is not ready.
- Do not introduce vector DB or hidden RAG memory.
- Do not copy reference project prompts or source code.
- Do not make reference recall a default step for every chapter.
- Do not write reference facts into `characters/`, `world/`, `timeline/`, `foreshadow/`, or `state/`.
