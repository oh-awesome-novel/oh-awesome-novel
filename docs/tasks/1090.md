## Goal

把 `1060` 已完成的 Play session core helper 接到实际产品面：用户可以创建 Play session、继续 transcript、查看 activated sources / observations，并把 adoption candidate 转成 PendingAction。

## Review Trigger

原任务以“Play panel or route”为交付边界，最初实现选择了 `WorkspaceRightPanel` 中的 `PlayModeTab.vue`。新的稳定产品约束已经明确：Play 与 Writing 是同层级的 workspace 顶级模式，右侧 Play panel 只能视为历史过渡实现。

顶级 UI、client、streaming / cancel 与 workspace-scoped mode / layout 恢复已经完成阶段性闭环。本任务仍保持 `Needs Review`，因为 selected branch / checkpoint 与完整浏览器级 Play 旅程尚未完成；这些增量由 `1120` 继续追踪。

- Play files 只写 `.workspace/play-sessions/<session-id>/`。
- world referee route 复用现有 single Aider-style runtime，不引入多 agent 平台。
- adoption candidate 必须携带 payload；后端只生成 PendingAction，不直接写 canonical truth。
- 覆盖测试：
  - `__test__/core/src/play-session.test.ts`

- 新 `PlayWorkspace` 使用独立 session rail、transcript、composer、world HUD、event feed 和 adoption panel；旧 `PlayModeTab.vue` 已删除。
- `workspaceMode` 与 Writing 工作台布局偏好按 workspace 持久化；重开 workspace 可恢复到 Play，当前 Play session 继续使用独立 workspace-scoped selection。session rail 已补充展开与当前项 ARIA 语义，并有 mounted / router 测试。
- Play session v4（含必需 branch base / legacy cutoff）、world-referee stream / cancel、scheduled hard-due 外部事件及后续 checkpoint / variant 由 `1120` 继续追踪。

## Deliverables

- Backend routes for creating, reading, updating, and listing Play sessions.
- Client API for Play session lifecycle.
- Top-level independent Play workspace, not a Writing right-panel tab.
- World referee prompt execution through the existing single Aider-style runtime, not a multi-agent platform.
- Activated source selection and display.
- Transcript / play-local state / observations persistence.
- Adoption candidate conversion to `chapter.createDraft`, `state.set`, `timeline.add`, or `foreshadow.create` PendingAction.
- Tests covering Play path safety, route behavior, and adoption boundary.

## Done Criteria

- Play session files are written only under `.workspace/play-sessions/<session-id>/`.
- Play transcript and play-local state are visible to the user but not canonical truth.
- User can create an adoption candidate and review it as a PendingAction.
- No canonical file changes until PendingAction accept.
- Imported Tavern interaction hints can be used in Play context without overriding constitution or canonical character facts.
- Workspace 顶级导航提供 `Writing | Play`，Play 切换主工作区而不是打开 `rightTab: 'play'`。
- `WorkspaceRightTab` / `WorkspaceRightPanel` 不再承载完整 Play Mode；Play 的 HUD / Event / Context / Adoption inspector 属于 `PlayWorkspace` 内部。
- 当前 Play session、selected branch 与 Play 布局状态可在模式切换和 workspace 重开后恢复。

## Constraints

- Do not introduce default heavy multi-agent runtime.
- Do not silently adopt Play observations into truth files.
- Do not let imported prompt overrides supersede OAN constitution.
- Do not fetch cards from remote sharing sites.
- Do not bypass Human Approval.
