## Goal

把已实现的 projection generator 和 project health model 接入 agent intake、PendingAction accept 后刷新、以及 UI guardrail。

Project health 和 projection 仍然是只读 / 派生能力，不成为事实源，也不阻断普通写作第一版。

## Current Code State

- `packages/core/src/projections.ts` 已能生成 `.oan/indexes/state.md`、`foreshadow.md`、`timeline.md`、`progress.md` 和 `context-snapshot.md`。
- `packages/core/src/project-health.ts` 已能读取缺失角色卡、未摘要章节、active hooks、stale state、timeline gaps 和 pending action count。
- `packages/backend/src/index.ts` 已有 `GET /api/workspace/project-health`。
- `packages/backend/src/index.ts` 已有 `POST /api/workspace/projections/rebuild` 显式 rebuild route。
- PendingAction accept / reject 响应已返回 refreshed workspace status 和 project health。
- model mode agent intake 已附带 concise project health guardrail context。
- Workspace UI 已展示 project health 摘要，并提供显式 Rebuild Projections 操作。
- `apps/desktop-ui/src/components/workspace/WorkspaceShell.vue` 会在 PendingAction decision 后应用 backend refresh payload。

## Implementation Notes

- projection rebuild 只写 `.oan/indexes/*` 生成文件，不替代 canonical Object File Tree。
- health guardrail 只作为 selected context / user-visible warning，不阻断普通写作。
- projection rebuild 失败只影响 rebuild action，不影响 PendingAction accept / reject。
- 覆盖测试：
  - `__test__/backend/src/backend.test.ts`
  - `__test__/client/src/client.test.ts`

## Deliverables

- Backend route to rebuild projections explicitly.
- Backend post-accept refresh payload that includes updated workspace status and project health.
- UI refresh flow after PendingAction accept / reject.
- Optional agent intake context item with concise project health warnings.
- Guardrail copy for stale state, missing settlement, timeline gaps, and unresolved PendingActions.
- Tests for projection rebuild route, post-accept refresh, and read-only guardrail behavior.

## Done Criteria

- User can explicitly rebuild projections.
- Projection rebuild writes only `.oan/indexes/*` generated files.
- PendingAction accept refreshes pending count, git status, project health, and optionally projection freshness.
- Agent can mention relevant project health warnings without treating projection as truth.
- Projection failure does not block normal writing unless the user explicitly requested projection rebuild.

## Constraints

- Projection is rebuildable derived output, not canonical truth.
- Do not let write-intent tools read projection instead of truth files when truth files are needed.
- Do not silently modify canon from dashboard / health checks.
- Do not introduce SQLite, vector read-model, or hidden cache as source of truth.
