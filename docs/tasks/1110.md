# 1110 Agent Usage And Context Stats

> Status: Planned
> Milestone: Agent Observability And Context Governance

## Related Plans

- [Agent Usage And Context Stats Plan](../AGENT_USAGE_AND_CONTEXT_STATS_PLAN.md)

## Goal

为 agent copilot 增加 message token、context token 和 AI SDK provider usage 统计。

这项任务的目的不是做复杂 observability 平台，而是让 OAN 能回答：

- 本轮 agent 实际送入了多少上下文。
- 哪些 context source 占用了主要 token budget。
- provider 返回了多少 input / output / total tokens。
- provider 没有返回 usage 时，系统仍能提供清晰的 estimated stats。

## Current State

当前已有基础：

- `ContextPackage` 记录 selected / omitted / reason / budgetLayer。
- runtime context builder 会把上下文转成 model-visible messages。
- AI SDK `streamText` 已提供 `usage`、`totalUsage`、`onFinish.totalUsage` 和 `onStepFinish.usage`。

当前缺口：

- OAN 没有 message-level estimated token stats。
- OAN 没有 context-level estimated token stats。
- OAN 没有把 AI SDK `totalUsage` 接入 runtime result。
- OAN 没有 session-level usage artifact。
- UI 没有展示 estimated / actual token usage。

## Deliverables

- message / context estimated token stats。
- AI SDK `streamText.totalUsage` capture。
- runtime `usage_stats` event。
- turn-level aggregated usage summary。
- session artifact：`.workspace/sessions/<session-id>/usage-stats.jsonl`。
- UI usage summary，明确区分 estimated / actual。
- tests 覆盖 provider missing usage fallback。

## Done Criteria

- 每次 model step 都能记录 estimated input stats。
- provider 返回 usage 时，step stats 包含 actual usage。
- provider 不返回 usage 时，actual usage 保持 unavailable，不伪造为 0。
- turn finish result 包含 aggregated usage summary。
- session artifact 可恢复查看 usage stats。
- UI 明确展示 estimated / actual 的区别。
- context package summary、summaries、state、timeline、foreshadow 等 context item 能按 kind 汇总 estimated tokens。
- 测试覆盖 core stats、runtime event、agent AI SDK usage capture、backend/UI stream data。

## Constraints

- 不引入 provider-specific tokenizer 或重型 tokenizer dependency。
- 不写入小说 truth files。
- 不改变现有 prompt assembly 顺序。
- 不阻塞 provider missing usage 的模型调用。
- 不把 estimated token 当作真实计费数据。
- 不引入重型 agent runtime 或 observability 平台。

## Recommended Implementation Order

1. 在 `packages/core` 增加 stats helper 和类型。
2. 扩展 runtime model response、turn result 和 `usage_stats` event。
3. 在 agent AI SDK adapter 中读取 `streamText.totalUsage`。
4. 将 usage stats 写入 session artifact。
5. 将 `usage_stats` 转成 UI stream data chunk。
6. 在桌面 UI 中做轻量 summary 展示。
7. 补齐 core / runtime / agent / backend 测试。
