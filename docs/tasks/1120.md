## Goal

让 Play Mode 不只追加角色对话，而是通过单一 world referee 在同一 Play-local 回合中结算叙事、时间、外部事件、状态变化、observation 和行动建议，同时保持 canonical truth 与 Human Approval 边界。

## Implemented Slice

- Play session schema v4：保留 `revision`、world clock、event policy、typed world events、suggested actions，并把结构化回合事实迁移到 `turns/<turn-id>.yaml`。
- `session.yaml` 只保存 metadata、`selectedTurnIds` 和必需的 `branchSnapshotRequiredFromRevision` / `branchBaseSnapshot`，不再保存完整 transcript；`transcript.md` 和 Core 暴露的内存 transcript 均从 selected turn path 单向投影，手工修改 projection 不会反向成为事实。
- settlement 与手工 transcript mutation 都会生成 turn artifact；artifact 可携带 parent、输入、消息、clock、event / observation refs、state delta、suggested actions 和 `canonical: false`。
- turn graph、selected root-to-head path、artifact / message / event / observation / adoption candidate 唯一性和跨 ledger 引用采用 fail-closed 校验；artifact schema v2 + session legacy cutoff 防止把新回合降级伪装成 legacy，严格 `artifactKind` 区分 settlement 与普通 transcript append。版本化 branch snapshot 同时封存 world clock、Play-local state value / visibility、suggested actions、完整 schedule head 与 hard-due evidence；v4 的 branch base 为 root 或 legacy bridge 提供可验证前驱。referee prompt、cause validation、feed 与 scheduled-event evaluator 只读取 selected projection，不能信任被篡改的内存 transcript 或 sibling branch snapshot。ledger 可保留 provenance 连贯的兄弟分支历史，但手工新建 observation / candidate 只能引用当时 selected branch，UI 也只投影当前 selected facts。
- committed turn artifact 已作为隐式 checkpoint 暴露；恢复通过 mandatory `baseRevision`、同 session lock 和 staged snapshot 切换 selected root-to-head path，同时恢复 transcript、world state / visibility、schedule 与 suggestions。session revision 保持单调，全部 artifact / event / observation / adoption ledger 保留；恢复后新回合会从目标 head 形成 sibling variant，旧后续回合可再次选择。checkpoint preview 只使用用户输入，不旁路泄露 referee / hidden narrative。
- 完整 `worldSettlement` artifact 已支持同一 before-turn snapshot 上的原子 Retry。独立 SSE route 严格只接受 mandatory `baseRevision`，行动与 action kind 从 source artifact 读取；Core 在内存中恢复目标前态并用一次 settlement 生成同 parent sibling，旧 source artifact / descendants 与全部 ledger 保留。普通 turn 与 Retry 共用 provider stream、run registry、Stop / reconcile、commit barrier，以及 session reservation、cooperative filesystem lock 与紧贴 commit barrier 的 revision recheck；取消、失败或检测到 drift 时零写入。首回合通过共享虚拟 branch base 的完整 v2 root forest 分叉，Desktop 在 provisional 期间投影 before-state、显示原始 action，并明确旧结果保留为 variant。
- v1 / v2 session 可兼容读取为 legacy turn artifacts；空的 v3 session 或 structured turns 仍为 artifact v1 且 projection 可验证的 v3 session 也可显式迁移。Core migration preview 会报告生成的 turn id、未知 metadata key 和备份位置。第一次写入 v4 时，原始 session 与 preview 一并进入 `.migrations/v1-to-v4/`、`.migrations/v2-to-v4/` 或 `.migrations/v3-to-v4/`，未知顶层 session metadata 被保留，migration history 在后续 staged save 中继续保留。升级时封存当前 selected legacy projection 为 `branchBaseSnapshot`，并以当前 revision 作为 cutoff；内含 artifact v2 但缺少两项证据的 v3 session 仍 fail closed。
- `events.yaml` 保存结构化 Play-local world events；`event-schedule.yaml` 保存 typed scheduled event、trigger、状态与 resolution evidence。旧 session 缺少事件或 schedule 文件时按兼容空值读取。
- 宿主已提供确定性的 `nextTurn` / `afterTurns` / `flagEquals` hard-due evaluator；`atWorldTime` 只有显式规范化比较器时启用，`manual` 不自动触发。到期事件必须由 settlement 以 `triggerId` 恰好结算一次，并且不受 spontaneous event budget 限制。
- referee 可通过 typed `scheduledEventChanges` 创建、取消或改期未来事件；宿主生成 schedule id、revision、turn 与 source evidence，schedule 与 turn / state / events 在同一 staged snapshot 提交。
- world referee prompt 装配最近 transcript、Play-local state、事件、world clock、policy 与 activated source 实际内容。
- M1 world momentum 已落地：strict Pressure / Agenda 保存在 reserved Play-local state 中并进入 branch snapshot；宿主在 selected before-state 上结合 action、typed relative wait、simulation mode 与 density 计算稳定 eligible candidates。settlement 只能通过 typed pressure / agenda changes 推进被引用的 eligible 动力，raw state delta 不能绕过 contract。
- Composer 已提供等待预设与自定义相对时间推进，HUD / Event Feed 展示 spoiler-aware active pressure、deadline、agenda next move 与安全 cause labels；Restore / Retry 会恢复同一份 momentum 与 typed wait evidence。
- referee 只能使用 read tools；不能在 Play 回合中直接调用 canonical write-intent tools。
- `oan-play-settlement` 结构化结算协议：正文与 JSON settlement 分离，宿主生成 turn/event id、sequence、timestamp 和 `canonical: false`。
- settlement block 为必需协议；无效 settlement、超预算事件和伪造 cause reference 在写入前失败，用户输入、referee 文本和状态均不会产生部分提交。
- session snapshot 先写 sibling staging directory，再通过 directory swap 提交；读取时可恢复带 `.ready` 标记的中断事务。
- 所有同 session Play-local mutation 共享进程内互斥锁与可选 `baseRevision` 检查，不再只锁 world turn。F1 correctness gate 已让 staged session writer / reader / recovery 与 rehearsal attempt recovery 使用 cooperative cross-process filesystem lock，并在持锁区间执行 authoritative revision CAS、attempt staged publish 和 Finish transaction；未经 Core writer 的任意外部原始文件改写仍不在协作锁保证内。
- 显式重复 session id、未来 schema version 和 workspace 外 symlink source 会被拒绝。
- client 已接通 world-referee endpoint。
- Play 专用 turn SSE 已接通 `started -> context.ready -> narrative.delta* -> (narrative.reset -> narrative.delta*)* -> prepared -> (play.event.occurred*) -> committed / cancelled / failed`；响应头预先暴露 turn id 供首帧前断流对账，中间 read-tool loop 使用 reset 清除旧 provisional block。provider response 会隔离到发现 settlement boundary 后才检查并释放纯 prose narrative prefix；无 fence、合法 fence 前的 raw JSON / settlement 字段、额外 fence 或原始 marker 都不产生 provisional delta / commit，失败 / conflict 会撤回未提交 provisional，只有 committed session 才能替换权威状态。
- staged snapshot swap 写入成功后，Backend 会在 terminal `play.turn.committed` 前发布逐项 `play.event.occurred`；cancel / validation / provider / commit failure 不发布。`play.turn.committed.session` 仍是 transcript、world state、schedule、observation 与 event feed 的权威 UI 刷新载荷。
- 服务端使用绑定不可变 workspace 的 turn registry、显式 cancel endpoint 与 commit barrier 处理 stop / cancel 竞态；活跃回合期间拒绝 workspace create / open / remove，commit 开始前取消不会推进 revision，commit 已开始后会明确返回 too-late / committed truth。
- Desktop UI 已提供真实 provisional block、Stop、取消/失败/冲突/结果未知状态与独立 `aria-live` 状态播报；发送期间锁定 session 切换、刷新和 adoption mutation，对账 transport 失败时不谎报“未提交”，而是要求刷新服务端事实后再继续。workspace mode 与工作台布局按 workspace 恢复，session rail 具备明确的展开和当前项 ARIA 语义。
- 顶级独立 Play workspace：session rail、transcript、action composer、world clock、event feed、spoiler toggle、state/source HUD、observation/adoption。
- v4 state visibility 必须精确覆盖 state 叶路径，observation / candidate 必须显式声明 visibility；缺失、非法或危险路径会 fail closed。hidden turn 的 state / observation / candidate 与 `cause.reason` 使用 author-only projection；spoiler 关闭时不会从 HUD、event cause 或 adoption 泄露，但允许隐藏原因产生公开可感知的后果。
- adoption 使用目标化表单生成 candidate，携带去重的 observation / turn / event provenance，再显式创建 PendingAction；candidate id 由宿主生成，不接受调用方控制；不再暴露 raw JSON 作为主要体验。

## Done Criteria

- 一个 Play turn 以结构化 draft 验证后一次提交，不存在 transcript-only 或 state-only 半回合。
- 外部事件有 kind、origin、cause、visibility、turn id、sequence 和 world clock 证据。
- hidden event 默认不进入 player-visible event feed，作者可显式开启 spoiler。
- Play referee 只读 canonical sources，Play 结果只能经 adoption candidate -> PendingAction 进入真实文件树。
- stale revision 在调用模型前返回冲突；无效 settlement 不改变 session。
- 旧 session 可继续读取和升级，不要求批量破坏式迁移。
- Core、Backend、Client 和 UI build 验证通过。

## Constraints

- 不引入多 Agent runtime、后台自治 Agent、SQLite 事实源或重型 event sourcing。
- 不在 `packages/runtime` 中加入 Play 领域逻辑。
- 不复制 SillyTavern、InkOS 或 OpenTavern 的代码与 prompt。
- 不允许 Play 回合直接写 canonical 文件。
