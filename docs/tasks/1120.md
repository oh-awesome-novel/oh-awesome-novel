# 1120 Play World Events And Turn Settlement

> Status: Needs Review
> Milestone: Play Mode

## Related Plans

- [Play Mode World Events Upgrade Plan](../PLAY_MODE_WORLD_EVENTS_UPGRADE_PLAN.md)

## Goal

让 Play Mode 不只追加角色对话，而是通过单一 world referee 在同一 Play-local 回合中结算叙事、时间、外部事件、状态变化、observation 和行动建议，同时保持 canonical truth 与 Human Approval 边界。

## Implemented Slice

- Play session schema v3：保留 `revision`、world clock、event policy、typed world events、suggested actions，并把结构化回合事实迁移到 `turns/<turn-id>.yaml`。
- `session.yaml` 只保存 metadata 与 `selectedTurnIds`，不再保存完整 transcript；`transcript.md` 和 Core 暴露的内存 transcript 均从 selected turn path 单向投影，手工修改 projection 不会反向成为事实。
- settlement 与手工 transcript mutation 都会生成 turn artifact；artifact 可携带 parent、输入、消息、clock、event / observation refs、state delta、suggested actions 和 `canonical: false`。
- turn graph、selected root-to-head path、artifact / message / event / observation 唯一性和跨 ledger 引用采用 fail-closed 校验；referee prompt 与 cause validation 只读取 selected projection，不能信任被篡改的内存 transcript。
- v1 / v2 session 可兼容读取为 legacy turn artifacts；Core migration preview 会报告生成的 turn id、未知 metadata key 和备份位置。第一次写入 v3 时，原始 session 与 preview 一并进入 `.migrations/`，未知顶层 session metadata 被保留，migration history 在后续 staged save 中继续保留。
- `events.yaml` 保存结构化 Play-local world events；旧 session 缺少既有字段或事件文件时按兼容默认值读取。
- world referee prompt 装配最近 transcript、Play-local state、事件、world clock、policy 与 activated source 实际内容。
- referee 只能使用 read tools；不能在 Play 回合中直接调用 canonical write-intent tools。
- `oan-play-settlement` 结构化结算协议：正文与 JSON settlement 分离，宿主生成 turn/event id、sequence、timestamp 和 `canonical: false`。
- settlement block 为必需协议；无效 settlement、超预算事件和伪造 cause reference 在写入前失败，用户输入、referee 文本和状态均不会产生部分提交。
- session snapshot 先写 sibling staging directory，再通过 directory swap 提交；读取时可恢复带 `.ready` 标记的中断事务。
- 所有同 session Play-local mutation 共享互斥锁与可选 `baseRevision` compare-and-swap，不再只锁 world turn。
- 显式重复 session id、未来 schema version 和 workspace 外 symlink source 会被拒绝。
- client 已接通 world-referee endpoint。
- Play 专用 turn SSE 已接通 `started -> context.ready -> narrative.delta* -> (narrative.reset -> narrative.delta*)* -> prepared -> committed / cancelled / failed`；响应头预先暴露 turn id 供首帧前断流对账，中间 read-tool loop 使用 reset 清除旧 provisional block，provisional narrative 通过增量 fence filter 与结构化 settlement 分流，只有 committed session 才能替换权威状态。
- 当前 `play.turn.committed.session` 是 transcript、world state、observation 与 event feed 的权威刷新载荷；独立 `play.event.occurred` 尚未作为 UI 事实源。
- 服务端使用绑定不可变 workspace 的 turn registry、显式 cancel endpoint 与 commit barrier 处理 stop / cancel 竞态；活跃回合期间拒绝 workspace create / open / remove，commit 开始前取消不会推进 revision，commit 已开始后会明确返回 too-late / committed truth。
- Desktop UI 已提供真实 provisional block、Stop、取消/失败/冲突/结果未知状态与独立 `aria-live` 状态播报；发送期间锁定 session 切换、刷新和 adoption mutation，对账 transport 失败时不谎报“未提交”，而是要求刷新服务端事实后再继续。
- 顶级独立 Play workspace：session rail、transcript、action composer、world clock、event feed、spoiler toggle、state/source HUD、observation/adoption。
- hidden turn 的 state / observation / candidate 共享 author-only visibility projection；spoiler 关闭时不会从 HUD 或 adoption 泄露。
- adoption 使用目标化表单生成 candidate，携带 observation / turn / event provenance，再显式创建 PendingAction；不再暴露 raw JSON 作为主要体验。

- 在现有 selected turn path 基础上实现 checkpoint / variant / retry，并同时恢复 state、events、schedule、knowledge 与 selected transcript path；当前尚不能创建、切换或重试分支。
- 增加 Play context trace、omitted source 和 canonical source drift / rebase / fork。
- 将 session list 拆成 summary + selected detail，并为长 transcript / event feed 增加窗口化。
- 扩展根目录 `__test__/desktop-ui`：当前已覆盖 stream reducer、cancel / commit race 和中性样式契约，仍需补组件交互、窗口化、checkpoint / variant、键盘 / ARIA 与浏览器级 smoke。

## Done Criteria

- 一个 Play turn 以结构化 draft 验证后一次提交，不存在 transcript-only 或 state-only 半回合。
- 外部事件有 kind、origin、cause、visibility、turn id、sequence 和 world clock 证据。
- hidden event 默认不进入 player-visible event feed，作者可显式开启 spoiler。
- Play referee 只读 canonical sources，Play 结果只能经 adoption candidate -> PendingAction 进入真实文件树。
- stale revision 在调用模型前返回冲突；无效 settlement 不改变 session。
- 旧 session 可继续读取和升级，不要求批量破坏式迁移。
- Core、Backend、Client 和 UI build 验证通过。

## Constraints

- 不引入多 Agent runtime、后台自治 Agent、SQLite 事实源或重型 event sourcing。
- 不在 `packages/runtime` 中加入 Play 领域逻辑。
- 不复制 SillyTavern、InkOS 或 OpenTavern 的代码与 prompt。
- 不允许 Play 回合直接写 canonical 文件。
