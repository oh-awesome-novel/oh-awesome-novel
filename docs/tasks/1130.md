## Goal

在现有顶级 Play 工作区内交付首个可玩的 Scene Rehearsal：用户通过 compact Scene / Cast / Review 创建新的 rehearsal session，按固定 actor queue 生成受 participant perception 限制的 provisional step，并使用 Accept、Retry、Finish、Cancel 完成一次可复核、可取消、一次原子提交的角色推演。

## Scope

- 新建 rehearsal session 使用 parent v5 与 versioned `scene-rehearsal.yaml`；普通 Quick Start 与既有 v1-v4 session 保持现状。
- 最小 Scene Contract、session-local participant ref、冻结 initial knowledge evidence、participant-specific perception 与 fixed actor queue。
- attempt recovery、actor step variant、NarrativeBlock、Accept / Retry / Finish / Cancel、attempt revision 与 idempotency receipt。
- actor step 使用单一 referee 和独立 provisional SSE；只传 participant perception，不提供 canonical read / write tools。
- Finish 复用现有 settlement、hard-due、revision recheck 与 staged session writer，一次提交 turn artifact 和 scene evidence。
- actor contribution 中的 `playerVisible` event 生成无 committed id 的 provisional `worldNotice`；Finish 后每个 step 以 `settlementEventRefs` 冻结自己的事件分段，公开 hard-due 则进入独立 `hostNarrativeBlocks`，不归因给最后一个角色。
- Desktop 提供 compact setup、actor queue、provisional / selected / committed 文案、四个控制与 keyboard / ARIA 基线。

## Out Of Scope

- 既有 v4 session 转换为 rehearsal；因此本 task 不实现 migration confirmation UI。
- F2 source-backed Guided Start、F3 outcome / writing attachment、F4 revise / insert / grant knowledge / overlay evaluator。
- 多 Agent runtime、后台自治角色、MuseAI P2 跨设备或视听沉浸能力。

## Done Criteria

- v4 无 sidecar 可继续读取；v4 + orphan sidecar、v5 缺 sidecar或配对不一致会 fail closed。
- Scene / Cast / Review 最终确认前不创建 session；新 rehearsal session 重开后 setup 与冻结知识不丢失。
- actor step 的结构化 perception 只包含对应 participant evidence；已知 forbidden ref 输出在进入 attempt 前被拒绝。
- Accept / Retry 只改变 recovery attempt，不推进 session revision；Retry 失败或 Stop 保留旧 variant。
- Cancel 对 transcript、clock、state、events、schedule 与 turn artifact 零写入。
- Finish 同时校验 attempt revision、selected head 与 session base revision，并只提交一次；hard-due 只结算一次。
- owning turn artifact 与 committed scene evidence 互相引用，Restore / variant 只投影 selected branch evidence。
- committed aggregate blocks 必须严格等于 `steps[].narrativeBlocks` 展开后追加 `hostNarrativeBlocks`；step / host notice 的 refs、顺序、visibility 和 `title: summary` 内容必须从 owning artifact 事件精确派生，`rumor` / `playerUnknown` 不得升级为 rehearsal notice。
- Quick Start、普通 Play turn、checkpoint / Retry、world momentum 与 adoption 回归通过。
- Core、Backend、Client、Desktop 测试与 Desktop production build 通过。

## Implementation Notes

### 2026-07-15

- Core 新增独立的 rehearsal schema、attempt state machine、typed settlement 聚合与 `.recovery/turn-attempts` store；普通 session 保持 v4，新建 Scene Rehearsal 使用严格配对的 parent v5、`scene-rehearsal.yaml` 与 `scenes/*.yaml`。turn artifact v3 通过 stable refs 双向拥有 committed selected-step / NarrativeBlock evidence，Restore 只投影 selected branch。
- F1 scene evidence v1 冻结 `steps[].settlementEventRefs`、必需的 `hostNarrativeBlocks` 与顶层 aggregate blocks：角色 notice 只覆盖该 step 的 owning、非 hard-due、`playerVisible` events；公开 hard-due 只由无 `speakerRef` / `sourceRefs` 的宿主 notice 承载。Core 与 Client 对重复表示做 deep equality、event visibility dominance、精确 event partition 与派生内容校验。
- attempt recovery 使用 session 级 filesystem mutex、隐藏 stage 原子发布和可重建 active marker；空目录、残留 stage、terminal / missing marker 与 lost Cancel response 均可恢复，同时保持单 active attempt。Play session staged writer 增加 cooperative cross-process filesystem lock 与持锁 CAS，读取 / recovery 不会抢先发布活跃 writer 的 stage。
- Agent actor stream 只接收 frozen perception allowlist、已观察的 selected NarrativeBlocks 与当前 world clock，不接收 workspace、完整 session / attempt、Director objective / risk 或未观察的 player-visible event；单一 referee 只接收完整 host prompt 且无 tools。Backend / Client 提供独立 step-run、attempt 与 committed-turn identity，以及 strict create / get / stream / Stop / Accept / Finish / Cancel、revision、idempotency和 receipt-first replay 契约。
- referee prompt 同时携带 Play-local visibility map，并明确把 Director objective / risk、其它 participant 私有知识、`playerUnknown` state / event / schedule / momentum 和 hidden cause 视为 referee-private；公开 event title / summary 只能描述对应 visibility 下可感知的结果。Player result 只读取 selected committed branch，nested state 先展开为 dotted leaves 后仅展示 exact `playerVisible`，缺失 / `rumor` / `playerUnknown` 与 `worldMomentum.*` 一律 fail closed。
- Desktop 在顶级 Play 工作区提供 Purpose → Scene → Cast → Review → Create → Attempt → Step → Accept → Finish 完整旅程；持久化结果可在重开后恢复，支持下一次 attempt，并对 `cancelling` / `committing`、断流、run id 丢失、active lookup 失败和同 key replay 使用 fail-closed recovery。创建、确认、取消与结果切换具备键盘焦点返回、`aria-live` 和 reduced-motion 基线，样式继续使用黑白 / 灰阶 tokens。
- 验证通过：Core 22 files / 184 tests、Agent 5 / 25、Backend 2 / 46、Client 2 / 54、Desktop 22 / 99；Desktop production build 与 `git diff --check` 通过。MuseAI P2、F2 source-backed Guided Start、F3 Writing Handoff 与 F4 advanced Director controls 仍明确留在后续任务。

## Constraints

- `.recovery/turn-attempts` 是 shadow recovery，不是 committed Play truth。
- actor step Stop、attempt Cancel 与 committed turn terminal truth 必须使用不同 identity / event。
- 不把 attempt 或 rehearsal 状态机继续堆入 `play-session.ts` / `usePlayWorkspace.ts`。
- 不允许 rehearsal 直接修改 canonical 小说文件。
