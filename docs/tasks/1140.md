## Goal

让用户从真实章节、角色、世界、时间线与状态文件构造可复核的开局包，通过 Sources、Entry、Identity、Cast、Review 五步向导创建 Immersive Journey 或 Scene Rehearsal，同时保留现有 Quick Start。

## Scope

- versioned Play Launch Package、source role / identity / hash 与 missing / stale diagnostics。
- preview / create Backend 与 strict Client API。
- `.workspace/play-setups/<setup-id>/setup.yaml` 非 canonical 持久化。
- 最终确认后才创建 v4 immersive 或 v5 rehearsal session。
- Desktop 五步 Guided Start、键盘 / ARIA / reduced-motion 与中性设计。

## Out Of Scope

- canonical rebase / fork 自动合并。
- setup 版本历史、批量升级、跨设备同步或 MuseAI P2 视听增强。
- F3 outcome / writing attachment 与 F4 advanced Director controls。

## Done Criteria

- preview 从真实 workspace source ref 生成 path / role / hash / bounded excerpt，且不写任何 setup 或 session 文件。
- create 与 session start 会重新校验 source；missing / stale 阻止启动。
- Guided Start 五步在最终确认前不创建 session truth，也不修改 canonical 文件。
- Scene Rehearsal 复用现有 v5 Scene Contract / frozen perception；Immersive Journey 复用 v4 world-referee flow。
- Quick Start 和所有未转换 v1-v4 session 无回归。
- Core、Backend、Client、Desktop 测试与 Desktop production build 通过。

## Implementation Notes

### 2026-07-16

- `packages/core/src/play-launch.ts` 冻结 schema v1 Launch Package：preview 对 workspace 内真实文件做 containment、size、UTF-8 与 raw-byte SHA-256 检查，生成有界 excerpt 与 missing / invalid / stale diagnostics；确认时重新检查并只把服务端重建的 evidence 写入 `.workspace/play-setups/<id>/setup.yaml`。
- `packages/core/src/play-launch-session.ts` 将确认后的 package 映射到既有 v4 Immersive Journey 或 v5 Scene Rehearsal；session 只在 `activated-sources.yaml` 保存 source identity / hash / role，`metadataExtensions.playLaunch` 只保留 setup provenance，不建立第二份 source map 或 world clock truth。
- Backend / Client 提供 preview、create、read、start API 与严格运行时 guard。setup read、guided session reopen、session start 和每次 Immersive referee context 都重新验证全部 1–24 个来源；上下文最多展开前 8 个，但其余来源不会跳过校验，并有明确 omission 提示。
- session staged writer 新增持跨进程锁后的 create-only guard；Quick 与 Guided 同 ID 并发创建只允许一个赢家，冲突返回 409 且不会覆盖已落盘 session。
- Desktop 将 session rail 收敛为列表，在主工作区提供 Purpose -> Quick / Guided 入口；Guided 按 Sources、Entry、Identity、Cast、Review 五步装配，只有 Review 最终确认才依次保存 setup 并创建 session。公开 Client 类型直接驱动 composable 与组件接线。
- 根目录测试覆盖 no-write preview、path / binary / oversize、hash 与 evidence tamper、stale / missing、1–24 source 全量重验、setup / session reopen、v4 / v5 bridge、Quick 回归、并发 create-only、五步向导和完整 `PlayWorkspace` wiring journey。
- 验证结果：Core 24 files / 201 tests、Backend 2 files / 53 tests、Client 2 files / 57 tests、Desktop 26 files / 115 tests 全部通过；Desktop production build 通过，仅保留既有 chunk-size warning。

## Constraints

- 不建立平行 source identity、world clock 或 canonical truth。
- 不让前端直接读取 filesystem；所有 source preview 通过 Backend / Core。
- 不引入多 Agent runtime、数据库或后台自治世界。
