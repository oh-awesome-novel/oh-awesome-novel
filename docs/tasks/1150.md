## Goal

从当前 committed selected branch 生成可核验、可按 Player / Director 投影的 Outcome Report，并让用户只把明确选择的条目作为非 canonical Writing Reference 附到单次写作请求；任何 canonical 采用仍继续经过 PendingAction、diff 与 Human Approval。

## Scope

- v4 Immersive Journey 与 v5 Scene Rehearsal 的 selected-branch evidence index、Outcome Report 与角色 footprint。
- 每个 outcome item 的 visibility、tag、turn artifact / message / event / source / evidence refs。
- `.workspace/play-sessions/<id>/reports/outcome.yaml` 结构化派生报告与 `outcome.md` 投影。
- `.workspace/writing-references/<attachment-id>.yaml` create-only attachment、stale 校验与 detach 审计。
- 当前写作请求显式选择 attachment id 时才进入 Agent context 与 trace。
- outcome item 分组生成 observation / adoption candidate，并复用现有 PendingAction 采用链路。
- Desktop Outcome panel、Player / Director lens、附件选择与无障碍状态反馈。

## Out Of Scope

- Scene Memory、M3 branch-local knowledge / reveal、F4 advanced Director controls。
- 新 canonical adoption target、自动采用、自动将全部 active attachment 注入 prompt。
- discarded / unselected variant 内容、模型私有推理或 chain-of-thought 持久化。

## Done Criteria

- Report 只引用当前 committed selected branch；sibling、discarded 与 attempt-local 草稿均被拒绝。
- Player projection 不返回 `playerUnknown` item，也不从隐藏 evidence 派生标题、label 或上下文文本。
- Report 与 attachment 在 selected path、session revision 或 source hash 漂移后 fail closed 为 stale。
- Writing Reference 未显式附到当前请求时零注入；发送后本次选择清空；detached attachment 保留文件但不能再消费。
- Outcome adoption 只有 PendingAction accept 后才会修改 canonical 文件，Restore 后旧分支 candidate 不能继续创建 PendingAction。
- Core、Agent、Backend、Client、Desktop 测试与 Desktop production build 通过。

## Implementation Notes

### 2026-07-16

- `packages/core/src/play-outcome.ts` 从 immutable selected turn path 构建 v4 / v5 evidence index，生成 versioned Outcome Report、角色 footprint、稳定 evidence fingerprint、Director Markdown projection 与彻底清除 branch / artifact / message / event / observation / evidence / source / participant refs 的 Player projection。
- `packages/core/src/play-writing-reference.ts` 提供 create-only、最多 24 项、64 KiB request context budget、report fingerprint / selected branch / source snapshot stale 校验与 detach 审计；staged session writer 只按显式 manifest 保留 `reports/outcome.yaml|md`，拒绝 unknown、orphan、symlink、非普通文件和超限报告。
- Backend / Client 增加 report create/read、outcome item adoption、attachment create/list/detach 与 strict transport。active attempt、revision、selected branch、source drift 和 Restore 后旧 candidate 均 fail closed；Player stale reason 与所有内部引用去标识化。
- Agent 只消费当前 `/api/agent/chat` 请求显式给出的最多 8 个 active attachment id，生成 `userSelectedContext` trace 与 noncanonical rule boundary；不扫描 workspace 中其他 active attachment，也不把内容写入 minimal memory。
- Desktop 按冻结组件图接入 v4 / v5 Outcome panel、同步 Player / Director lens、24 项 attachment selection、审计 detach、candidate -> PendingAction 与 Copilot one-shot selector。Director -> Player 会在网络请求完成前本地清除隐藏数据；发送失败保留选择，成功、新建或切换对话后清空。
- 验证结果：Core 26 files / 217 tests、Backend 3 / 55、Client 3 / 60、Agent 5 / 28、Desktop 30 / 125 全部通过；Desktop production build 通过，仅保留既有 chunk-size warning。

## Constraints

- `selectedTurnIds` 是 turn artifact id；现有 observation / candidate `sourceTurnIds` 是 transcript message id，必须使用不同字段并由 evidence index 显式转换。
- Outcome 是可重建 projection，Writing Reference 是请求级 context attachment；二者都不是 canonical story truth。
- Session staged writer 必须显式保留并校验 `reports/`，不能依赖未知目录偶然存活。
