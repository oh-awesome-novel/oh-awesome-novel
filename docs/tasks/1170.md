## Goal

把 selected branch 中值得保留的 event、observation 或 F3 outcome item 用少量操作转换为 evidence-backed adoption candidate，先预览合法 canonical 目标与实际 diff，再进入既有 PendingAction / Human Approval 流程。

## Scope

- event / observation / outcome item 的统一 adoption draft 与 evidence closure。
- `chapterDraft | state | timeline | foreshadow` 合法目标建议、可编辑 payload 与目标说明。
- 创建 PendingAction 前的 canonical target preview 与 diff preview。
- selected-branch、visibility、source drift 与 Restore / Retry 后 candidate revalidation。
- Desktop Event Feed / Observation / Outcome 的一致“带回创作”入口、键盘操作和状态反馈。

## Out Of Scope

- 新 canonical adoption target、自动采用、自动接受 PendingAction。
- discarded / unselected variant、hidden provenance 的 Player label / preview。
- SemanticPatch 引擎重做、通用审批 UI 重构或 Play 外功能。

## Done Criteria

- selected event / observation 可在少量操作内得到目标建议、编辑预览与 PendingAction diff。
- candidate 保留 turn / event / observation / source evidence，且所有 provenance 属于当前 selected branch。
- Player label、payload 默认值与 diff preview 不从 hidden evidence 派生；Director 关闭后同步清除敏感草稿。
- PendingAction 创建前 canonical 文件不变；只有 accept 后写入，reject 后不写入。
- Restore / Retry、revision / source drift 或隐藏 evidence 失效后，旧 draft / candidate 不能继续创建 PendingAction。
- Core、Backend、Client、Desktop 测试和 Desktop production build 通过。

## Implementation Notes

### 2026-07-16

- `packages/core/src/play-adoption.ts` 冻结 event / observation / outcome 三类 identity-only `PlayAdoptionSeed`，只从当前 selected branch 与 current Outcome Report 重建完整 evidence closure；closure 绑定 session revision、selected path、turn / message / event / observation / rehearsal evidence 与 activated-source snapshot。四个既有 target 使用稳定建议与安全默认 payload，Player projection 会拒绝 hidden root，并重算彻底去引用后的 closure / fingerprint。
- write-intent tools 增加统一 `prepareWriteIntentPreview` / `validateWriteIntentPreview` / `promoteWriteIntentPreview` seam。preview 只生成 `.workspace` shadow、真实 target files 与 diff，不创建 PendingAction、不修改 canonical；确认先做无状态 preflight，promotion 再校验 fingerprint、normalized args / patches、shadow hash 与 canonical baseline，并拒绝重复 promotion。
- Backend 增加 server-owned adoption preview record 与 identity-only preview / promotion routes。确认时重新验证 active attempt、session revision、selected branch、Outcome fingerprint、source drift 与 canonical baseline；最终 accept 还会在跨进程 session filesystem lock 内重验 stored preview、candidate、live PendingAction、revision、branch 与 source。evidence-backed candidate 不再允许绕回旧 candidate-to-PendingAction route；candidate 先保持 non-canonical，PendingAction 幂等创建。Director 可查看真实 diff；Player 只收到不含 canonical baseline 的 proposal diff、投影后的 candidate 与最小 session revision receipt，真实 PendingAction 仍只在显式 Review / approval surface 中处理。
- Client 对 seed、closure、suggestion、payload、target file、diff、PendingAction 与 refresh 响应执行 strict unknown-field / enum / identity guard；Outcome API 显式返回 report fingerprint，供 outcome seed 精确绑定当前报告。
- Desktop 的 Event、Observation 与 Outcome 三入口统一进入 `PlayAdoptionDraftForm`：展示 server suggestion、四 target、可编辑 JSON、按 Lens 安全投影的 target / diff、canonical 边界与显式 Review。lens、session、revision、Restore / Retry 或 source context 变化会同步清除草稿和晚响应；确认成功后即使 Lens / session 已切换，也只做非敏感 revision reconciliation 与全局 PendingAction refresh，不把旧 Director candidate 投回 Player。Play 内不提供 accept / reject。
- 验证结果：Core 28 files / 231 tests、Tools 7 / 58、Backend 4 / 69、Client 5 / 91、Agent 5 / 28、Desktop 34 / 139 全部通过；相关 package build、Client TypeScript 检查与 Desktop production build 通过，仅保留既有 chunk-size warning。

## Constraints

- 复用现有 `PlayAdoptionCandidate`、write-intent tools、PendingAction diff 与审批 API。
- 目标建议只能在现有四个合法 target 内选择；不得暗中扩展 canonical schema。
- candidate 与 preview 都是 non-canonical；Play 发生过不等于小说事实。
