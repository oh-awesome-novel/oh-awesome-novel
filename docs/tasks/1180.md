## Goal

让长篇 Play session 不再通过列表或主工作区一次传输和渲染全部历史，并让每个已提交回合都能解释实际使用、遗漏和发生漂移的 source；用户可在 canonical source 变化时明确选择继续当前冻结上下文、重新装配或 fork，而不是被静默混入新事实。

## Scope

- versioned `PlaySessionSummary`、selected detail 与 transcript / event cursor window。
- summary list 不返回完整 transcript、event、turn artifact、observation 或 adoption ledger。
- host-owned turn-scoped Play context trace：记录 transcript / event window、selected / omitted source、预算原因、content hash、规则边界与 drift 状态，不保存模型私有推理。
- 成功提交的 turn 在同一次 staged snapshot 中保存对应 `traces/*.context.yaml`；取消、provider / validation / CAS 失败零写入。
- source drift status 与三种 typed 决策：继续当前冻结上下文、从当前 canonical 重新装配、fork 新 session。
- Desktop summary rail、窗口化 transcript / event feed、context/source inspector 与长列表体验。
- 一条真实 renderer / browser-level Play 长旅程回归，并收口被新契约替代的旧读取入口。

## Out Of Scope

- F4 rehearsal-specific Director interventions、Scene Memory 或 participant knowledge grant。
- 通用 fsync / power-loss matrix、可重启 terminal registry、provider 通用 backpressure 重构。
- 自动修改 canonical source、自动接受 PendingAction 或保存 chain-of-thought。
- 旧 endpoint 下线、兼容迁移与已发布数据 migration；产品发布前允许旧入口暂留。
- screen-reader、完整焦点旅程与系统化无障碍验收；本阶段只保留基础原生控件语义。

## Done Criteria

- session rail 只消费 summary；选中 session 后按 cursor 加载 transcript / event window，旧页可追加且不会重复或跨 revision 混合。
- cursor 绑定 session、selected branch 与 revision；Restore / Retry / 新 turn 后旧 cursor fail closed 并触发权威刷新。
- 每个成功 world-referee turn 的 trace 能列出实际 transcript / event 范围、source hash、selected / omitted 状态与预算原因。
- Player projection 不泄漏 hidden source 内容；Director inspector 只显示结构化证据，不显示模型私有推理。
- drift 决策均为显式、typed、可审计操作；继续当前内容不会静默加载变化后的 canonical bytes，重新装配 / fork 不覆盖旧历史。
- 长 session 核心功能旅程有真实 renderer/browser smoke。
- Core、Backend、Client、Desktop 测试和 Desktop production build 通过。

## Implementation Notes

### 2026-07-20

- `packages/core/src/play-session-read-model.ts` 增加 summary、selected detail 与绑定 session / revision / selected head / window kind 的 opaque cursor；主列表不再返回 transcript、event、artifact、observation 或 adoption ledger。bounded detail 同时返回与 event window 一一同序的 Player-safe / Author evidence，以及只描述 selected head 的最小 artifact presentation，使 v5 rehearsal 重开时无需恢复完整 artifact ledger。
- `packages/core/src/play-context-trace.ts` 把实际 transcript / event window、source selection / omission、预算原因、hash 与可验证 drift state 保存为 host-owned evidence。world turn 的 trace 与 artifact 进入同一 staged session snapshot；取消、provider / validation / CAS 失败零写入，同 artifact trace 不可改写。
- `packages/core/src/play-source-drift.ts`、Backend 与 strict Client 落地 `continueFrozen | reassemble | fork`。决策使用 session revision CAS、canonical race 复核与 active-attempt 冲突保护；fork 以固定字典序同时锁定 source / target，使权威读取、drift 双检与 staged target commit 处于同一跨进程事务窗口。当前阶段采用冲突即 409 的 fail-closed 语义，不额外承诺 decision receipt replay。
- Desktop rail 只消费 summary，选中后由 `usePlaySessionHistoryWindow.ts` 管理 bounded detail、早期 transcript / event 拼接、stale cursor 权威刷新、context inspector 与 drift control。事件分页会同步合并 bounded presentation evidence，Player projection 不读取 Author evidence；v5 selected artifact presentation 直接交给 rehearsal projector，不伪造完整 turn artifact。Electron smoke 通过 Vite test harness 直接编译并挂载真实 `PlayWorkspace.vue`，覆盖 summary 选择、两类分页、trace / drift 控制与 F4 typed Modify，不再以静态 HTML fixture 代替产品路径。
- 旧 full-session endpoint 仅在显式 Retry graph operation 中惰性读取；旧 endpoint 下线、兼容 migration 与已发布数据迁移按产品早期阶段要求暂缓。系统化 screen-reader / 完整焦点矩阵同样不属于本 task，只保留基础原生控件语义与功能 renderer smoke。
- 验证结果：Core 252、Agent 28、Backend 75、Client 96、Desktop 161、Runtime 10、Tools 58，共 95 个测试文件 / 680 项测试全部通过；Electron renderer smoke、support TypeScript 检查、相关 package build、Desktop production build 与 `git diff --check` 通过。仅有既存 fsmonitor 与 Vite chunk-size 非阻塞警告。

## Component Map

```text
PlayWorkspace.vue
  -> usePlaySessionHistoryWindow.ts (selected detail + cursor orchestration)
  -> PlaySessionRail.vue (summary-only list; emits selection/refresh)
  -> PlayTranscript.vue (presentational window; emits load-older)
  -> PlayEventFeed.vue (presentational window; emits load-older)
  -> PlayContextInspector.vue (trace/drift presentation; emits typed decision)
```

- `PlayWorkspace.vue` 只做 API / composable 装配，不持有窗口算法。
- 两个 feed 组件只渲染已投影 window，不自行访问 API。
- inspector 只接收安全 view model，以 typed event 上送用户选择。

## Constraints

- `turns/*.yaml`、selected path、events ledger 与 Play-local state 继续是事实源；summary、window 与 trace 是可验证读模型 / 证据，不建立第二套 turn truth。
- 所有 `.workspace/play-sessions` 写入继续复用 staged session writer、filesystem lock 与 revision CAS。
- 保持现有 v4 Immersive Journey、v5 Scene Rehearsal、Quick Start、Restore / Retry 与 adoption 兼容。
- 测试继续放在根目录 `__test__/` workspaces。
