## Goal

在现有 Scene Rehearsal attempt、selected step variant、单一 world referee 与一次 Finish 提交上，增加严格 typed 的修改、重定向、插队和 participant knowledge grant；较早步骤变化会保留旧 variant、失效受影响后缀并重算 provisional overlay，而不会破坏 committed history。

## Scope

- versioned、append-only `PlayDirectorIntervention`：`reviseProjection | redirectStep | insertActor | grantKnowledge`。
- projection-only revise 必须保持 effect fingerprint；redirect 必须重新裁决。
- earlier-step fork、suffix invalidation、step variant preservation 与 selected head 更新。
- 复用现有 due evaluator 的纯 provisional overlay adapter；不得建立第二套 schedule / event reducer。
- `directorFixed | refereeDynamic | hybrid` actor order、合法 `noMaterialEffect` 与连续停滞提示。
- participant-scoped branch-local knowledge grant，包含 stable evidence / author provenance，并从指定 step 生效。
- visibility-aware、可重建且可 stale 的 Scene Memory。
- knowledge / behavior anchor / event inspector、Player / Director Lens 与七个 Director 用户控制。

## Out Of Scope

- 多 Agent runtime、后台自治角色或按现实时间推进世界。
- 修改旧 committed artifact、复制 checkpoint / variant truth 或把 Scene Memory 变成事实源。
- 自动采用 rehearsal 结果到 canonical 小说文件。
- 旧格式 migration、兼容转换与 endpoint 下线；产品发布前只维护当前开发态格式。
- screen-reader、完整焦点旅程与系统化无障碍验收。

## Done Criteria

- revise / redirect / insert / grant 请求只接受冻结的 typed payload；未知字段、自由 `unknown` effect 与 stale attempt revision fail closed。
- revise 只替换安全 narrative projection，effect fingerprint 不变；redirect 从目标 before-step 状态生成新 variant，并使旧后缀 superseded。
- insert 更新 attempt actor order 并失效受影响后缀；dynamic / hybrid 顺序仍由宿主在 Scene Contract 允许范围内确定。
- participant grant 只改变指定 participant 的 perception，带 provenance，并随 attempt variant、Finish、Restore 与 Retry 隔离。
- overlay evaluator 与 committed evaluator 对相同 projected snapshot 的 due ordering / trigger 结果一致；hard-due 最终只提交一次。
- `noMaterialEffect` 是合法 typed result；连续停滞只提示 Director，不自动制造冲突。
- Scene Memory 只从 committed selected branch 重建，revision、selected path 或 source hash 变化后 stale。
- 七个控制均接入 typed mutation；Player Lens 不泄漏 hidden evidence。
- Core、Agent、Backend、Client、Desktop 测试和 Desktop production build 通过。

## Implementation Notes

### 2026-07-20

- `packages/core/src/play-turn-attempt.ts` 冻结 schema v1 append-only `reviseProjection | redirectStep | insertActor | grantKnowledge`，包含 attempt revision、idempotency receipt、effect fingerprint、variant / suffix invalidation、`directorFixed | refereeDynamic | hybrid` scheduler、typed `noMaterialEffect` 与非自动冲突的 stagnation 提示。projection revise 保持宿主 effect closure，并原样保留 UI 未展示的 Director-only / player-unknown blocks。
- `packages/core/src/play-rehearsal-overlay.ts` 只构造 disposable selected-prefix projection，继续调用既有 state merge、schedule materializer 与 due evaluator；Finish 使用同一 overlay due 集，actor contribution 不能伪造 hard-due，最终仍只提交一次 world settlement。
- `packages/core/src/play-rehearsal-knowledge.ts` 与 M3 `playKnowledge` participant grant transition 接通：Director grant 在 Finish 后写入 selected branch snapshot，Restore / Retry 随 branch truth 恢复；后续 perception 只向目标 participant 提供生效 evidence，不能提升 event 全局 visibility。
- `packages/core/src/play-scene-memory.ts` 从 committed selected branch / Outcome evidence 重建可丢弃 projection，绑定 revision、selected path 与 source hashes；memory 会随 session staged directory swap 安全保留，读取时在同一 session transaction 内重算校验，变化后标记 stale，而不是静默消失。Player memory 清除 selected turn、source、event、evidence 与 participant identities，Backend 只返回对应 lens 的投影。
- Backend 的公开 redirect 只接受 step ref、Director intent 与 constraint refs，由宿主重新运行 actor 和唯一 world referee；constraint ref 必须属于目标角色 before-step perception 中的 initial / 已生效 grant / visible event / observed block 闭包，provider 调用前由 Backend 拒绝悬空或跨分支引用，Core seam 再次 fail closed。Referee 只接收宿主解析并安全序列化的 typed constraint record；公开请求不能注入 replacement step、perception 或 effects。Client 对四类 mutation、participant grant、attempt receipt 与 memory 执行 unknown-field / enum / revision / collection-bound strict validation。
- Desktop 提供 `Accept | Modify | Retry | Insert actor | Grant knowledge | Finish | Cancel` 七控制、projection-only / redirect 双修改协议、typed forms、suffix / stagnation 提示、Player / Director inspector 与 Scene Memory current / stale / rebuild。Player Lens 会同步清除 Director memory，并按 fact / grant visibility 投影 participant knowledge。
- 旧 endpoint / 旧格式 migration 与系统化 screen-reader / 完整焦点旅程按产品早期阶段要求暂缓，不阻塞本 task。验证结果与 M5 合并回归一致：95 个测试文件 / 680 项测试、Desktop production build、真实 `PlayWorkspace.vue` Electron renderer smoke、support TypeScript 检查与 `git diff --check` 全部通过。

## Component Map

## Constraints

- F4 必须复用现有 attempt recovery、settlement validator、hard-due evaluator、session staged writer、turn graph、checkpoint / variant 与 M3 knowledge store。
- append-only intervention / variant history 不得物理覆盖旧 step。
- `.recovery/turn-attempts` 仍是未提交 shadow；`memories/` 仅为可重建 projection。
- canonical 写入继续经过 PendingAction / diff / Human Approval。
- 测试继续放在根目录 `__test__/` workspaces。
